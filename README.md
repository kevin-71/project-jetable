# Sonora

Monorepo de départ pour la bibliothèque de podcasts auto-générés décrite dans `sonora.md`.

## Dossiers
- `sonora-frontend/`
- `sonora-backend/`
- `sonora-playlist-service/`
- `sonora-audio-service/`

## Lancement
1. Copier les variables d'environnement depuis `.env.example` dans chaque service si besoin.
2. Lancer `docker compose up --build` à la racine.

Ce dépôt contient une ossature prête à compléter avec les implémentations métier et les intégrations externes.
